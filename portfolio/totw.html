TAILS OF THE WILD

C# & UNITY || PERSONAL

An infinite top-down rogue-like made between 2019 and 2020

This game currently features an infinite run, in which the player must make their way to the exit of each randomly generated level without starving, or running out of health. Enemies constantly pursue the player in an attempt to hinder progress.

A generated level, viewed from the Unity inspector.

In particular, I focused on the random level generation aspect, inspired by the random aspects of tabletop games such as Gloomhaven. The game determines a random number of rooms, R, to generate, and continues to generate them in a limited space, giving them a random width, height, and number of door tiles with each attempt. It ensures that these rooms do not overlap, nor that they escape the level boundary. Then, it randomly shuffles these rooms, generating corridors, C, between them until all rooms are connected and inactive. All generation is initially completed as an array of strings, which is then used to actually build the level using the wall and floor prefabs created in Unity, which are each given a random texture for variety. On top of this, enemies, E, and items, I, are spawned in the level to attack the player, and to heal the player's hunger or health. Finally, the player itself, P, is given a starting position that does not overlap with an enemy.

Enemies chasing the player

This system is based on a separate coordinate system which allows enemies to use a variant of an A* search algorithm to track down and attack the player, or run away to random tiles in the map depending on the random behaviour chosen. The enemy determines where to move using a reference to the whole map as a string. After a set amount of steps, an enemy reevaluates their next move by randomly changing their behaviour. Enemies would continue to generate as the level continues, away from the player, until they eventually find them to attack.
