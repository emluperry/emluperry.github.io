CLEMMINGS

Unity & C# || COURSEWORK

Clone Lemmings - a simple 3D clone of a classic resource management puzzle game.

In the first semester of my second year, I had to create a game focusing around any one mechanic in Unity. After discussion with one of my lecturers, I landed on trying to create a clone of Lemmings. For each level, I created a GameManager object that would keep track of level specific variables, such as the number of Lemmings the player starts with and should end the game with, the number of each role the player can assign to Lemmings in the level, and so on. I then used an object pool pattern to create instances of each Lemming GameObject, as well as other creatable objects such as the Lemming explosion particle effect, while each level is loading in order to reduce the impact of lag during runtime.

Lemmings exiting a level.

I also put a lot of effort into decoupling systems in this game. This posed a challenge while developing systems such as UI, animation, and sound, so I decided to use an event based system using the observer pattern. When a Lemming's physics script determines that it has died, it fires an event that these systems listen to and update depending on which function it is listening to.

While developing this game, I also took the time to become more familiar with additive scene loading and Unity's mixer system, which proved useful for tracking persistent objects between levels, such as the scene/UI manager or settings across the game. Now that I am more familiar with these systems, I hope to make better use of them in future projects as it proved particularly useful! If I ever work further on it, it will likely be to test certain uses of UI - I never needed to add a second page to the level select screen, so something as simple as that could be done here, if I don't just do it in a future project.
