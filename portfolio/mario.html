MARIO RECREATION

C++/SDL || COURSEWORK

A simple recreation of the first two levels of Super Mario Bros for the NES.

A recreation of the first two levels of the Super Mario Bros NES game, created over the course of my second semester while developing my coding skills using SDL. The game features functioning copies of the first two levels, with enemies that update while onscreen and die when stomped by the player or hit by a POW block shockwave. Only a few features are missing from the original game.
Levels are generated using a tile system, in which characters in a text file correlate to a block type, which is then displayed on screen. It uses a similar system for enemy generation, able to place different enemies in a grid based on the character used. To complement this, the background scrolls. The appearance of textures in level 1 and 2 are different, and able to be set by changing one variable.

All moving entities in the game make use of inheritance and polymorphism from a base Character class, which is inherited by both the players (Mario and Luigi) and all enemies, such as Koopas and Goombas. Similarly, all tiles in the game inherit from one Tile class, with child classes as Bricks, POW Blocks and Concrete which have different properties to determine whether or not they are breakable.
Breaking blocks and defeating enemies awards points to the player, which are tracked on the top left of the screen.
